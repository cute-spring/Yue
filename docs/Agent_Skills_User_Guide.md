## 1. 引言与设计目的

随着 Yue 项目演进的复杂化，我们在 `docs/plans/` 目录下积累了大量的架构演进文档（Epic）。为了解决 AI 在跨会话时“遗忘上下文”的问题，以及人工管理 Backlog 状态繁琐的痛点，我们引入了**AI 技能化 (Skill-based) 工作流**。

本指南旨在帮助研发团队熟练掌握自定义 Skill（特别是最新创建的 `epic-starter` 和 `plan-archiver`），将 AI 从单纯的“代码补全器”升级为你的“专属项目经理 + 架构师”，实现开发流程的极度自动化。

---

## 2. 新技能详解：自动化项目管理双擎

### 2.1 Epic Starter (一键启动上下文)
*   **设计目的**：解决 AI 开新局时没有上下文的痛点，让 AI 自动阅读总控看板，精准定位当前开发任务。
*   **核心能力**：
    *   自动读取并解析 `docs/plans/INDEX.md`。
    *   深入读取目标 Epic 的 Markdown 详情。
    *   智能识别未完成的 `[ ]` 任务节点。
    *   输出结构化的“开工报告”并建议代码修改方案。
*   **适用场景**：每天早上开始工作时；接手别人写了一半的架构演进计划时；切换开发任务时。

### 2.2 Plan Archiver (一键自动归档)
*   **设计目的**：解决任务完成后，文档乱堆、看板状态不同步的问题，维持项目目录的绝对整洁。
*   **核心能力**：
    *   自动将完成的 Markdown 文档移动到 `archive/` 目录。
    *   自动在 `INDEX.md` 中将任务状态流转到 `Done` 区域并修复链接。
    *   自动执行 `git add` 和 `git commit`。
*   **适用场景**：某个 Epic 或 Phase 的所有子任务全部完成，需要结项收尾时。

## 3. 使用步骤与实战演示

### 场景一：开始开发一个新功能
假设你要开始做数据库演进计划。

**Step 1: 唤醒 AI 并触发技能**
打开一个新的 AI 对话窗口，直接输入以下指令：
> “调用 `epic-starter` 技能，我要开始执行数据库演进计划。”

**Step 2: 审核 AI 的开工报告**
AI 会静默读取文件，然后输出如下报告：
```markdown
# 🚀 Epic Starter 已就绪
**已加载上下文：** 数据库架构演进
**目标文档：** docs/plans/Database_Evolution_Plan.md
**当前阶段：** Phase 1 - 引入 ORM 框架
**代码现状分析：** 当前 config_service.py 中使用的是 SQLite 本地路径。
**下一步行动方案：** 我将在 requirements.txt 中添加 SQLAlchemy。
请确认是否开始？
```

**Step 3: 确认执行**
你只需回复：“确认，开始写代码吧。”

### 场景二：计划执行完毕后的归档
当数据库演进的 4 个 Phase 都做完了。

**Step 1: 触发归档技能**
在当前对话或新对话中输入：
> “计划做完了，调用 `plan-archiver` 技能帮我归档 Database_Evolution_Plan.md。”

**Step 2: 享受自动化**
AI 会自动执行文件移动、修改 `INDEX.md`，并告诉你：
> “✅ 已成功将文档移动到 archive/ 目录，并完成 Git 提交 (commit: docs: archive completed plan)。”

## 4. 高级使用技巧与最佳实践

1.  💡 **不要使用模糊指令**：触发技能时，尽量明确说出“调用 [技能名]”，这比让 AI 自己去猜要稳定得多。
2.  💡 **结合 Git 保护技能**：如果你在主分支 (`main`) 误操作了，系统自带的 `git-main-guard` 技能会自动拦截你的 commit 并帮你建新分支。
3.  💡 **保持 INDEX.md 的纯洁**：`epic-starter` 高度依赖 `INDEX.md` 的结构。请保持其中 `[ ]` (待办) 和 `[x]` (完成) 语法的标准性。
4.  💡 **单次对话单任务**：不要在一个对话里让 AI 同时启动两个 Epic。AI 的上下文窗口有限，一次专注一个计划能产出最高质量的代码。
5.  💡 **定期碎片整理**：每月发一次指令：“请帮我梳理 docs/plans 下的文档并更新 INDEX.md”，让 AI 帮你做项目管家。

## 5. Yue 核心技能库清单 (Skills Inventory)

除了新加的两个技能，Yue 项目还内置了丰富的生产力 Skill。以下是现有核心技能的标准化介绍：

### 🔧 研发自动化与保护类

**1. git-main-guard (主分支守卫)**
*   **功能概述**：拦截直接在 `main` 或 `master` 分支上的 `git add/commit` 请求。
*   **使用场景**：当开发者忘记切分支，直接在主分支写完代码并要求 AI 提交时。
*   **关键说明**：触发后，它会自动根据你的 commit 意图生成一个 Feature 分支名（如 `feat/add-login`），切换过去后再执行提交。

**2. feature-staged-delivery (特性分阶段交付)**
*   **功能概述**：强制推行企业级的“分阶段发布”方法论，防止大爆炸式上线。
*   **使用场景**：当要求 AI 开发一个涉及核心链路（如 LLM 流式输出、高频数据库写压）的复杂功能时。
*   **关键说明**：它会强迫 AI 按照“基线对齐 -> 仅输出日志(静默) -> 灰度上线”的严谨节奏写代码。

### 🏗️ 架构与运维评估类

**3. production-readiness-auditor (生产就绪体检)**
*   **功能概述**：一键生成企业级软件或 AI 系统的“生产就绪健康报告”。
*   **使用场景**：项目即将上线前，要求 AI 评估“目前系统还有哪些坑没填”。
*   **关键说明**：会从架构、安全、容灾等 10+ 维度输出结构化的 Gap 分析和风险评级。

**4. mcp-builder (MCP 服务构建指南)**
*   **功能概述**：指导 AI 或开发者如何编写高质量的 Model Context Protocol (MCP) 服务器。
*   **使用场景**：当需要将外部 API（如 Jira, 数据库）封装为大模型可用的 Tool 时。
*   **关键说明**：包含 Python (FastMCP) 和 TypeScript 的标准模板与最佳实践。

### 🎓 技能开发与学习类

**5. skill-creator (技能创建引擎)**
*   **功能概述**：指导 AI 如何编写一个新的 Skill。
*   **使用场景**：当团队需要把某个重复性工作流固化为新技能时。
*   **关键说明**：本指南中的 `epic-starter` 就是通过此技能生成的。

**6. skill-mastery-coach (技能精通教练)**
*   **功能概述**：交互式的教练，帮助开发者学习如何设计优秀的 Prompt 和 Skill。
*   **使用场景**：新员工入职，想学习如何更好地驾驭 AI Agent 时。

### 🛠️ 特定场景与扩展工具类 (Extended/WIP)

**7. ppt-expert (PPT 制作专家)**
*   **功能概述**：提供基于 Markdown 或特定框架快速生成结构化演示文稿的能力。
*   **使用场景**：当开发团队需要快速生成技术架构汇报、Sprint 总结报告的幻灯片骨架时。
*   **关键说明**：这是一个偏向文档办公的技能，建议在技术方案设计完成后，结合已有的架构文档触发此技能生成汇报材料。

**8. rca-expert (根因分析专家) [待完善]**
*   **功能概述**：旨在帮助团队在发生线上故障（Incident）后，通过结构化的 5-Whys 或其他方法论快速进行 Root Cause Analysis。
*   **使用场景**：线上 P0/P1 故障恢复后，团队需要复盘并输出事故报告时。
*   **关键说明**：当前版本可能需要补充具体的项目错误码字典和日志分析脚本才能发挥最大威力。

**9. visual-inspection (视觉检测工具) [待完善]**
*   **功能概述**：用于 UI 自动化测试或图像对比，分析前端页面的视觉回归问题。
*   **使用场景**：前端组件库升级或大规模 CSS 重构后，验证页面是否发生非预期的样式错位。
*   **关键说明**：通常需要配合具体的视觉回归测试框架（如 Playwright/Cypress 的截图对比功能）结合使用。

**10. agent-tool-architect (Agent 工具架构师) [待完善]**
*   **功能概述**：协助规划和设计供大语言模型使用的高级工具（Tool/Function Calling）的输入输出 Schema 和错误边界。
*   **使用场景**：在开发新的后端 API 供 Agent 调用前，用于设计最符合 LLM 认知的工具协议。
*   **关键说明**：该技能的部分生态位与 `mcp-builder` 重合，建议在设计复杂业务工具组时结合使用。

## 6. 常见问题解答 (FAQ)

**Q1: 如果 AI 在执行 `epic-starter` 时找错了文档怎么办？**
**A1**: 可以在指令中直接写明路径：“调用 epic-starter，加载 `docs/plans/Logging_Config_Evolution_Plan.md`”。

**Q2: `plan-archiver` 归档后，我在其他文档里引用了这个计划的链接失效了怎么办？**
**A2**: 目前 `plan-archiver` 只会自动修复 `INDEX.md` 中的链接。对于散落在其他代码注释中的链接，建议使用全局搜索替换功能。

**Q3: 我可以自己修改这些技能的逻辑吗？**
**A3**: 当然可以！所有的技能都存放在代码库的 `.trae/skills/` 目录下。你只需修改对应技能文件夹下的 `SKILL.md`，修改后立即生效。
