# PDF Built-in Tools 高 ROI 专题文档

本文聚焦 PDF 相关内置工具的高 ROI 能力，并按“纯 Python 可落地 vs 需要外部组件”重新分类，给出对照表、最小依赖路径与落地方案。

---

## 1. 当前后端依赖基线

已存在 PDF 库：
- **pypdf**（纯 Python，已在后端依赖中）

当前可直接依赖的能力以文本层为主，缺少 OCR 与复杂版式/图像层处理。

已落地的 PDF 相关能力（内置工具）：
- **docs_search_pdf**：在 PDF 中搜索关键词，默认每个文件最多扫 6 页（可配置），返回命中片段与页码范围
- **docs_read_pdf**：按页读取 PDF 内容，默认最多读取 6 页（可配置）
- **pdf_outline_extract**：提取 PDF 目录/书签结构 (Phase 1)
- **pdf_keyword_page_search**：在指定页码范围内快速检索关键词 (Phase 1)
- **pdf_page_text_read**：提取指定页面的纯文本 (Phase 1)
- **pdf_page_range_filter**：过滤有效页码集合 (Phase 1)
- **pdf_page_table_extract**：利用 `pdfplumber` 提取页面中的结构化表格 (Phase 2)
- **pdf_page_render_image**：利用 `PyMuPDF` 将页面渲染为图片 (Phase 2)

适用范围与限制：
- 依赖 pypdf 的文本层抽取，适合文本型 PDF
- 不支持 OCR 与图像型 PDF 内容识别

## 3. 工具详细说明（原子化能力）

### 3.1 PDF 目录提取工具
**Function Name**：pdf_outline_extract  
**Input**：PDF 路径/文件  
**Output**：目录标题、对应页码、层级结构  
**作用价值**：直接获取文档元数据中的书签结构。对于有规范目录的文档（如年报、研报），可瞬间完成章节定位，避免无意义的全文扫描。

### 3.2 关键词页码检索工具
**Function Name**：pdf_keyword_page_search  
**Input**：关键词数组、页码范围、PDF 路径/文件  
**Output**：包含关键词 of 页码数组  
**作用价值**：基于文本层扫描，快速锁定包含特定术语的页面。支持在限定页码范围内搜索，极大地提高了检索效率。

### 3.3 PDF 页面表格提取工具
**Function Name**：pdf_page_table_extract  
**Input**：PDF 路径/文件、页码、表格序号  
**Output**：表格行列结构化数据  
**作用价值**：利用启发式算法识别页面中的表格边框和对齐关系，将其转换为 Python 列表或字典。解决 PDF 中表格数据“看得见抽不出”的问题。

### 3.4 指定页码文本提取工具
**Function Name**：pdf_page_text_read  
**Input**：PDF 路径/文件、页码  
**Output**：该页纯文本内容  
**作用价值**：最基础的读取能力。将 PDF 格式的二进制数据流还原为人类可读的字符串，是所有文本分析工作的起点。

### 3.5 页码范围过滤工具
**Function Name**：pdf_page_range_filter  
**Input**：起始页、结束页、候选页码集合  
**Output**：过滤后的有效页码集合  
**作用价值**：纯逻辑处理工具。用于配合目录定位后的二次裁剪，确保后续的高耗时操作（如表格提取或全页扫描）仅作用于有效范围。

### 3.6 PDF 页面转图片工具
**Function Name**：pdf_page_render_image  
**Input**：PDF 路径/文件、页码、分辨率（dpi）  
**Output**：图片文件路径或字节数据  
**作用价值**：**核心支柱**。在文本抽取失败、版式极其复杂或处理扫描件时，为多模态 LLM（如 Gemini/GPT-4o）提供直观的视觉参考，极大地弥补了纯文本抽取的局限。

## 5. 纯 Python 可落地能力详述

### 3.1 结构化目录抽取
**用途**：输出章节层级与页码映射，支撑按章节阅读、跳转、分块与定位。  
**核心输出**：章节树（层级、标题、起止页、父子关系）。  
**落地要点**：优先读取 PDF Outline；无目录时退化为标题启发式。

### 3.2 文本抽取与分页读取
**用途**：按页抽取文本，作为其他能力的基础输入。  
**核心输出**：页码 + 文本。  
**落地要点**：限制页数与字节上限，避免超大文件拖慢处理。

### 3.3 分块切片器
**用途**：按页/段落/标题层级切片，形成检索与摘要最小单元。  
**核心输出**：`chunk_id`、`page_range`、`section_path`、`text`。  
**落地要点**：提供页级与标题启发式切片。

### 3.4 元信息抽取（标题/作者/日期）
**用途**：抽取文档元信息用于分类与检索。  
**核心输出**：标准化元数据字段。  
**落地要点**：先读文档信息字典，再做首屏启发式识别。

### 3.5 引用/参考文献抽取
**用途**：结构化输出参考文献与引用条目。  
**核心输出**：引文列表。  
**落地要点**：定位“References/参考文献”章节，再分条抽取。

### 3.6 跨页合并修复
**用途**：修复断行、断段、连字符，合并跨页文本。  
**核心输出**：连续文本与合并标记。  
**落地要点**：基于标点与断行规则拼接。

### 3.7 内容指纹与去重
**用途**：对页/块生成 hash，用于缓存与增量更新。  
**核心输出**：`hash` 与 `chunk_id` 映射。  
**落地要点**：确保 hash 稳定、与块结构强绑定。

### 3.8 关键词与主题标签抽取
**用途**：输出文档/片段主题标签。  
**核心输出**：关键词列表。  
**落地要点**：使用规则 + 简单统计权重。

## 6. 最小依赖路径（建议）

### 4.1 基线（当前可直接落地）
- **pypdf + 标准库**
- 覆盖能力：目录、元信息、文本分页、分块、引用抽取、跨页修复、指纹与标签

### 4.2 纯 Python 增强（需新增依赖）
- **PyMuPDF / pypdfum2**：实现页面渲染与转图片，赋能视觉模型  
- **pdfminer.six**：提供字符/块坐标，支持版式解析与页内锚点  
- **pdfplumber**：基于 pdfminer.six 的轻量表格抽取  
- 覆盖能力：页面渲染、版式噪声清理、页内锚点定位、基础表格抽取

### 4.3 外部组件增强（非纯 Python）
- **Tesseract / PaddleOCR**：OCR 与置信度  
- **Camelot + Ghostscript / Tabula + Java**：复杂表格  
- **OpenCV + 模型**：图表识别与标注抽取

## 8. 总结
本版文档保留纯 Python 可落地能力为主线，并提供外部组件的可选扩展路径。建议以 pypdf 为基线快速落地第一阶段，再视需求引入 pdfminer.six/pdfplumber，最后按业务价值引入 OCR 与图像处理能力。
