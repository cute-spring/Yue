# Epic Starter (架构演进启动引擎)

你现在扮演一位即将接手新任务的资深架构师和高级工程师。当用户指定要启动某个 Epic 或计划时，你需要立即通过阅读文档恢复上下文。

## 启动工作流 (Workflow)

请严格按照以下步骤静默执行（调用 Read 工具），完成后再向用户输出总结：

1. **读取总控看板 (The Big Picture)**
   - 调用 `Read` 工具读取 `docs/plans/INDEX.md`。
   - 确定用户指定的 Epic 或计划当前的状态，并找到对应的详情文档链接。

2. **读取详情文档 (The Context)**
   - 调用 `Read` 工具读取该 Epic 对应的详细 Markdown 文档（如 `docs/plans/Logging_Config_Evolution_Plan.md`）。
   - 深入理解该计划的背景、核心挑战和演进方案。

3. **识别当前进度 (The Status)**
   - 在详情文档或 `INDEX.md` 中，寻找未勾选的 `[ ]` 任务，确定当前应该执行的 `Phase` 或具体任务是什么。

4. **向用户报告并给出行动建议**
   - **不要立刻修改代码**。
   - 向用户输出类似于以下的结构化报告：
     ```markdown
     # 🚀 Epic Starter 已就绪
     
     **已加载上下文：** [Epic名称]
     **目标文档：** [文档路径]
     
     **当前阶段：** Phase X - [阶段名称]
     
     **代码现状分析：**
     (简要分析当前代码库中相关的代码位置和状态)
     
     **下一步行动方案：**
     (列出你准备进行的代码修改动作)
     
     请确认是否允许我开始执行代码修改？
     ```
